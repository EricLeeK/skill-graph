SkillGraph 的核心洞察是「预处理智能」：一次性使用 LLM 深入理解技能、生成合成查询并构建智能索引，运行时完全基于预建索引进行毫秒级检索，无需任何 LLM 调用。系统在 SkillsBench 的 87 项任务上达到 72.1% Recall@10，超越 UCSB Agentic 基线（68.3%），同时实现零 Token 消耗与 170 倍速度提升。

核心创新：LLM-for-Index

将 LLM 的推理能力从运行时前移至预处理阶段，实现「一次投入，永久受益」的检索范式。

  • 为每个技能生成 10 条合成用户查询，覆盖多样表达
  • 多向量索引同时存储技能嵌入与合成查询嵌入
  • 运行时仅依赖向量相似度计算，零 LLM 调用
  • 合成查询带来 +9.3pp Recall 提升，代价仅为一次性的预处理 Token

技术架构：SACR + 多向量索引

三层社区层次结构实现由粗到精的渐进式过滤，平衡召回率与延迟。

  • SACR 路由：8/64/512 三级社区层次，候选集从 34K 缩减至 500-2K
  • 语义重排：基于 all-MiniLM-L6-v2 的余弦相似度精细排序
  • 混合评分：max(技能嵌入相似度, 最佳合成查询相似度)
  • 端到端延迟 17.5ms，纯 SACR 过滤仅需 ~10ms

关键结论：SkillGraph 证明「用 LLM 建索引，而不是用 LLM 做检索」是一种可扩展的范式。在 34K 真实技能上，它实现了比 LLM-based 方法更高的召回率、更低的延迟、以及零运行时成本。
