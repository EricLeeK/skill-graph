## 1. Project Overview

SkillGraph is a universal skill retrieval module for LLM-based agents (ReAct, LangChain, AutoGPT, etc.). It combines sparse attention-based community routing with LLM-generated synthetic queries to efficiently retrieve relevant skills from a large collection of 34,000+ real-world agent skills.

### Core Innovation: LLM-for-Index + Zero-Token Runtime

The project's key insight is **preprocessing intelligence**:
- **Preprocessing stage** (one-time, token-acceptable): Use LLM to deeply understand skills, generate synthetic user queries, build smart indices
- **Runtime stage** (zero-token, millisecond-level): Retrieve based on built indices

Current components:
1. **SACR (Sparse Attention Community Routing)**: Hierarchical KMeans clustering + query-aware top-k filtering
2. **Synthetic Query Generation**: LLM generates diverse user queries per skill, embeddings stored in multi-vector index
3. **Semantic Ranking**: Cosine similarity re-ranking from multi-vector space

### Architecture

```
Query
  |
  +---> SACR Router (sparse community filtering) ----+
  |                                                   |
  +---> Multi-Vector Index (skill_emb + syn_emb) ---+ |
  |                                                   |
  +---> Top-k Skills                                  |
  |                                                    |
  +---> SRE Refinement (optional)                     |
```

## 4. Core Components

### 4.1 SACR - Sparse Attention Community Router

- Routes queries through 3-level community hierarchy (8/64/512 communities)
- Parameters (`top_k_level0/1/2`) are system configs calibrated on target dataset
- Pure SACR: 59.4% Recall@10, ~10ms latency
- Candidate set: typically 500-2000 skills (of 34,396)

### 4.2 Synthetic Query Generation

- Input: 34,396 skills' name + description
- Output: 10 synthetic queries per skill (343,960 total)
- Generated by LLM (Kimi agent cluster) with user-perspective prompts
- Embedded with all-MiniLM-L6-v2, stored in multi-vector index

### 4.3 Multi-Vector Retrieval

```python
# Skill embedding similarity
skill_sim = cosine(q_emb, skill_emb)

# Best synthetic query similarity
syn_sims = cosine(q_emb, syn_embeddings[skill_id])  # (10,)
best_syn_sim = max(syn_sims)

# Combined score
score = max(skill_sim, best_syn_sim)
```

## 5. Current Benchmark Results

### SkillsBench (87 tasks, 34K UCSB skills)

| System | Recall@10 | Latency | Token/Query |
|--------|-----------|---------|-------------|
| **Dense + SynQ** | **72.1% ± 16.2%** | **17.5 ms** | **0** |
| Dense | 62.8% ± 12.7% | 2.8 ms | 0 |
| SACR | 59.4% ± 9.2% | 10.0 ms | 0 |
| SACR + SynQ | 72.1% ± 16.2% | 26.5 ms | 0 |
| UCSB Agentic | 68.3% | ~seconds | ~5000 |

**Key findings**:
- SynQ provides +9.3pp improvement over dense baseline
- Dense + SynQ exceeds UCSB Agentic (68.3%) with zero runtime tokens
- 170x faster latency than UCSB's LLM-based approach
